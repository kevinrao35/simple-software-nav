OBS Studio 使用教程：从录屏到直播完全指南

进阶软件：OBS Studio

从下载安装到专业直播，手把手教你用 OBS 录制和直播。

下载与安装

1 打开官网 obsproject.com，点击首页的 "Download" 按钮。

2 选择对应的操作系统版本：

- Windows：下载安装程序，运行后勾选"Browser Source"和"Virtual Camera"组件，这些后期可能会用到。

- macOS：下载 .dmg 文件，将 OBS 拖入 Applications 文件夹。首次启动需要授予屏幕录制权限。

- Linux：通过包管理器安装 sudo apt install obs-studio（Ubuntu）或 sudo dnf install obs-studio（Fedora）。

3 首次启动时，OBS 会自动运行自动配置向导，选择"优化推流"或"仅录制"，按指引完成初始设置。

> 包管理器：Windows 可用 winget install OBSProject.OBSStudio，macOS 可用 brew install --cask obs。

核心概念：场景与来源

OBS 的界面分为主播端和内容端。理解两个核心概念是使用 OBS 的基础：

### 场景（Scenes）

场景是画面的整体布局。你可以创建多个场景用于不同的场景切换：

"欢迎"场景——摄像头 + 背景图

"游戏"场景——游戏画面 + 摄像头

"休息"场景——休息提示图

"结束"场景——结束感谢画面

点击左下角的"+"按钮添加场景，在直播/录制中通过快捷键或点击场景名称切换。

### 来源（Sources）

来源是场景中的各个元素。每个场景可以叠加多个来源，上层的来源会覆盖下层。OBS 内置 10+ 种来源类型：

| 来源类型 | 用途 |

|---------|------|

| 显示器捕获 | 捕获整个屏幕画面 |

| 窗口捕获 | 捕获指定应用程序窗口 |

| 游戏捕获 | 捕获 DirectX/OpenGL 游戏画面，性能最好 |

| 图像 | 显示静态图片（Logo、背景、边框等） |

| 图像幻灯片 | 自动轮播图片 |

| 文本 | 显示文字（滚动字幕、标题等） |

| 媒体源 | 播放视频/音频文件 |

| 浏览器 | 嵌入网页（弹幕、直播面板等） |

| 视频捕获设备 | 摄像头/采集卡输入 |

| 音频输入/输出 | 麦克风/系统声音 |

操控技巧：

来源叠加顺序=画面图层顺序，拖拽调整。

选中来源后拖动调整位置和大小，红色边框表示选中。

`Ctrl+F` 快速适配屏幕，`Ctrl+S` 拉伸至全屏，`Ctrl+D` 居中。

混音器：音频控制

混音器面板显示所有音频来源的音量条：

每个音频设备显示实时音量波动。

点击音量条旁的喇叭图标一键静音。

点击齿轮图标→"滤镜"进入音频处理设置。

支持最多 2 个桌面音频 + 3 个麦克风/辅助设备。

### 常用音频滤镜

噪声抑制：消除背景噪音（风扇、空调声），推荐 RNNoise 算法。

噪声门：低于设定音量的声音直接静音，消除键盘声等。

增益：提升或降低整体音量。

压缩器：让音量更平稳，防止声音忽大忽小。

转场效果

在两个场景之间切换时，OBS 默认使用淡入淡出（Fade）。你也可以添加其他转场：

1 点击"转场"旁边的"+"按钮。

2 选择转场类型：

- Cut：直接切换，无过渡效果。

- Fade：淡入淡出，适合大多数场景。

- Swipe / Slide：滑动切换，适合演示场景。

- Fade to Color：渐变为纯色（黑场过渡）。

- Luma Wipe：亮度擦拭，创意转场。

3 设置转场时长（默认 300ms，建议 300-500ms）。

### 演播室模式（Studio Mode）

点击界面右下角的"演播室模式"按钮开启。这是专业直播的核心功能：

左侧：编辑区——你在后台修改场景、调整布局，观众看不到。

右侧：直播区——观众实际看到的内容。

点击中间的"Transition"按钮一键将编辑区内容推送到直播区。

观众完全察觉不到你在背后编辑。

推流设置

### 快速推流到平台

1 点击 设置 → 推流。

2 在"服务"下拉菜单中选择推流平台（B站/抖音/YouTube/Twitch 等）。

3 填入推流密钥（从直播平台获取）。

4 点击确定，回到主界面点击"开始推流"。

### 推流参数推荐

| 分辨率 | 码率 | 适用场景 |

|--------|------|---------|

| 1080p 60fps | 6000-8000 kbps | 游戏直播、专业直播 |

| 1080p 30fps | 4500-6000 kbps | 桌面直播、教学分享 |

| 720p 30fps | 2500-3500 kbps | 网速较差时保底方案 |

### 自定义推流（RTMP/SRT）

如果需要推送到自定义 RTMP 服务器：推流设置中选择"自定义..."，填入服务器 URL 和推流密钥。

OBS 还支持 SRT 和 RIST 协议，适合需要更低延迟和更稳定传输的场景。

录制设置

### 基础录制配置

1 设置 → 输出 → 录制。

2 选择录制质量：

- 与推流相同：录制的视频等于观众看到的画质。

- 高质量：独立于推流质量，录制更清晰的版本。

- 无损：巨大的文件体积，极高的画质。

3 选择编码器：

- 软件编码器（x264）：画质好，CPU 占用高。

- 硬件编码器（NVENC/AMF/QSV）：速度快，CPU 占用低。

### 回放缓冲区

回放缓冲区可以捕捉"刚才发生的精彩瞬间"：

1 设置 → 输出 → 回放缓冲区，启用并设置缓冲时长（30-120秒）。

2 设置 → 快捷键，为"保存回放"绑定快捷键（推荐 `Ctrl+F1`）。

3 游戏中按保存快捷键，自动保存过去 30 秒到文件。

> 小技巧：录制教程时建议选择"与推流不同"的质量设置，用 NVENC 编码器录制 1080p 60fps，确保录像清晰。

视频设置

基础（画布）分辨率：素材可放置的空间，通常匹配显示器分辨率（1920x1080）。

输出（缩放）分辨率：最终推流/录制的分辨率，常见 1080p 或 720p。

FPS 值：30（节省资源）或 60（流畅画面）。60fps 推流对 CPU/GPU 消耗远高于 30fps。

虚拟摄像头

OBS 的虚拟摄像头功能让 OBS 的画面可以被其他应用调用：

1 工具 → 虚拟摄像头，点击"启动"。

2 在腾讯会议/Zoom/钉钉等应用的摄像头设置中，选择 "OBS Virtual Camera"。

3 这样你就可以把 OBS 制作的画面（含叠加、滤镜、切换效果）作为摄像头输入源。

快捷键配置

设置 → 快捷键 中可以绑定以下常用操作：

| 操作 | 推荐快捷键 |

|------|-----------|

| 开始/停止推流 | Ctrl+F1 |

| 开始/停止录制 | Ctrl+F2 |

| 保存回放 | Ctrl+F3 |

| 切换到场景 N | Ctrl+1 到 Ctrl+9 |

| 麦克风一键静音 | Ctrl+Shift+M |

| 按键通话 | 按住设定键时说话，松手静音 |

高级功能：滤镜与特效

### 视频滤镜

为每个来源添加滤镜（右键来源 → 滤镜）：

色度键（Chroma Key）：绿幕去背景，把摄像头背景换成任何画面。

色彩校正：调整亮度、对比度、饱和度、色相。

裁剪/填充：裁剪多余画面区域。

滚动：制作滚动文字效果。

锐化：让文字/画面更清晰。

图像遮罩：圆形/异形摄像头边框。

### 浏览器来源

添加"浏览器"来源可以在 OBS 中嵌入网页：

直播弹幕：嵌入平台的弹幕 HTML 页面。

直播面板：显示观众信息、赞助提醒。

音乐可视化：嵌入网页音频可视化工具。

插件与脚本

OBS 支持通过 Lua 和 Python 脚本扩展功能：

工具 → 脚本：加载社区脚本或自己编写。

常见插件功能：音频可视化、自动场景切换、实时翻译等。

Windows 插件通过 `.dll` 文件安装，复制到 OBS 的 `obs-plugins` 目录。

常见问题

### 推流卡顿/丢帧怎么办？

1 降低输出分辨率（1080p → 720p）。

2 降低帧率（60fps → 30fps）。

3 降低码率（6000 kbps → 3500 kbps）。

4 切换硬件编码器（NVENC/AMF），减少 CPU 压力。

5 检查网络上行带宽是否足够（speedtest.net 测速）。

6 关闭不必要的程序，释放系统资源。

### 录制的视频文件太大？

1 使用 H.265 编码器录制，体积比 H.264 小 30-50%。

2 降低录制分辨率和码率。

3 录制完成后用 HandBrake 二次压缩。

### 笔记本黑屏/捕获不到游戏？

笔记本通常有两个 GPU（集成显卡 + 独立显卡）。OBS 需要在独立显卡上运行：

1 Windows 设置 → 图形设置 → 添加 OBS → 选择"高性能"。

2 使用"游戏捕获"而非"显示器捕获"捕获游戏画面。

3 禁用笔记本的"自动 GPU 切换"功能。

### 如何录制系统声音和麦克风并存？

OBS 默认会同时录制桌面音频（系统声音）和麦克风音频。确认混音器中两个音频源都已启用且音量正常。如果需要分别处理，建议在高级输出模式中录制多音轨：一个音轨存系统声音，一个音轨存麦克风，后期再分别调整。
