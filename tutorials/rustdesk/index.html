RustDesk 使用教程：免费远程桌面神器

进阶软件：RustDesk

下载与安装

1 打开官网 rustdesk.com，点击顶部的 "Download" 按钮。

2 选择对应的操作系统版本：

- Windows：下载 .exe 安装包，双击运行。普通用户选"仅为当前用户安装"，需要远程协助两端都安装。

- macOS：下载 .dmg 文件，拖入 Applications 文件夹。首次运行需要在系统偏好设置中授予辅助功能权限。

- Linux：下载 .deb（Ubuntu/Debian）或 .rpm（Fedora/CentOS）包，或通过 Flatpak 安装：flatpak install flathub com.rustdesk.RustDesk。

- Android：从 F-Droid 搜索 "RustDesk" 下载，或从官网下载 APK。

- iOS：从 App Store 搜索 "RustDesk" 安装。

3 安装完成后打开 RustDesk，主界面会显示当前设备的 ID 和 密码。

> 便携版：Windows 用户也可以下载便携版（.exe 单文件），无需安装即可运行，适合临时远程协助场景。

基本使用：远程控制

### 被控端（被控制的电脑）

1 启动 RustDesk，主界面显示当前设备的 ID 和临时密码。

2 将 ID 和安全密码告诉控制端（临时密码每次重启都会变化）。

3 可在 设置 → 安全 中设置固定密码，方便长期远程控制同一设备。

### 控制端（主动控制的电脑）

1 启动 RustDesk，在"远程桌面"选项卡输入被控端的 ID。

2 点击"连接"按钮。

3 输入被控端显示的密码，点击"确定"。

4 连接成功后，你可以看到对方的桌面并远程操作。

> 无需注册：RustDesk ID 直连，无需注册账号，无需登录，开箱即用。

核心功能详解

### 远程桌面

全屏模式：连接后按 `Ctrl+Alt+F` 进入全屏，获得接近本地操作的体验。

自适应窗口：远程桌面窗口自动适配本地窗口大小。

分辨率调整：支持缩放、适配屏幕等显示模式。

剪贴板同步：本地和远程设备之间的剪贴板内容自动同步。复制文本后可在对方电脑直接粘贴。

锁定键传递：支持 Ctrl+Alt+Del 等组合键传递到远程设备。

### 文件传输

1 连接远程桌面后，点击工具栏的文件传输图标（或按 `Ctrl+Alt+F` 切换模式）。

2 文件传输界面分为左右两栏：左侧是本地文件系统，右侧是远程文件系统。

3 选中文件直接拖拽即可上传或下载文件。

4 支持批量文件和文件夹传输。

### TCP 隧道

TCP 隧道功能允许你通过远程网络访问特定端口：

1 点击 网络 → TCP 隧道。

2 设置本地监听端口和远程目标地址。

3 访问本地的监听端口，流量会通过隧道转发到远程设备。

适用于：远程访问公司内网的 Web 服务、数据库等。

自建中继服务器

RustDesk 默认使用官方公共服务器。如果你对隐私或性能有更高要求，可以自建服务器。RustDesk 分为两个服务器组件：

| 组件 | 功能 | 端口 |

|------|------|------|

| hbbs | 信令服务器，负责设备发现和 ID 注册 | 21115-21117 |

| hbbr | 中继服务器，直连失败时转发数据 | 21118-21119 |

### Docker 一键部署（推荐）

docker run --name hbbs -d --restart=always \

-p 21115-21117:21115-21117/tcp \

-p 21116:21116/udp \

rustdesk/rustdesk-server hbbs

docker run --name hbbr -d --restart=always \

-p 21118-21119:21118-21119/tcp \

rustdesk/rustdesk-server hbbr

### 配置客户端使用自建服务器

1 在 RustDesk 客户端中点击"设置"。

2 找到 网络 选项卡。

3 在 中继服务器 中填写你的服务器地址和端口（默认 21116）。

4 点击"应用"后，客户端会自动连接自建服务器。

自建服务器后，所有远程连接流量走你自己的服务器，数据不经过任何第三方，速度和安全性都显著提升。

安全机制

RustDesk 在安全方面提供了多层保护：

端到端加密：所有远程会话使用加密通信，防止中间人攻击。

密码验证：每次连接需要输入被控端生成的临时密码或预设的固定密码。

白名单：设置 → 安全中可以配置允许连接的设备列表。

一键锁定：被控端点击"锁定"，需要重新授权才能远程控制。

自建服务器：数据完全由你掌控，不经过第三方服务器。

跨平台使用

RustDesk 支持在所有主流平台间互相远程控制：

| 控制端 ↓ / 被控端 → | Windows | macOS | Linux | Android | iOS |

|:---:|:---:|:---:|:---:|:---:|:---:|

| Windows | ✓ | ✓ | ✓ | ✓ | ✓ |

| macOS | ✓ | ✓ | ✓ | ✓ | ✓ |

| Linux | ✓ | ✓ | ✓ | ✓ | ✓ |

| Android | ✓ | ✓ | - | ✓ | ✓ |

| iOS | ✓ | ✓ | - | ✓ | ✓ |

Android 和 iOS 同样可以作为控制端，在手机上远程控制电脑。

常见问题

### RustDesk 是免费的吗？商用也可以吗？

RustDesk 是开源软件（AGPL-3.0 协议），个人和商业使用都免费。RustDesk Server Pro（含高级管理功能）需要付费，但自建 hbbs/hbbr 服务完全免费。

### 连接速度很慢怎么办？

慢通常是因为通过公共中继服务器转发。解决方法：

1 启用 直连：双方在同一局域网时自动直连，速度最快。

2 自建中继服务器：在你的云服务器上部署 hbbs + hbbr，连接走自己的服务器。

3 检查网络：如果不在同一个局域网，双方的上传带宽会影响连接质量。

### 连接时提示"连接失败"或找不到设备？

1 确认双方都运行了 RustDesk。

2 确认 ID 输入正确（区分大小写）。

3 检查防火墙：RustDesk 需要 UDP 端口 21116 可通。

4 如果使用了自建服务器，确认服务器运行正常。

5 有些企业网络会屏蔽 P2P 连接，尝试关闭 VPN 或换个网络环境。

### 怎么设置固定密码？

设置 → 安全 → 安全密码，输入并确认密码即可。设置了固定密码后，远程连接时不再需要临时密码。建议设置强密码（8 位以上，含字母和数字）。

### RustDesk 和 AnyDesk 哪个好？

RustDesk 最大的优势是开源和自建服务器能力。如果你的使用场景对数据隐私要求高，或者需要频繁远程办公，RustDesk 更合适。AnyDesk 的优点是商业支持和官方服务器全球节点多，不用自己维护服务器。

### 如何在 Linux 服务器上使用（SSH 无用的情况下）？

RustDesk 也提供命令行版本（需要显示器或虚拟显示器），可以通过 SSH 启动后，用 RustDesk 客户端连接图形界面：

sudo apt install ./rustdesk.deb

rustdesk --get-id

rustdesk --set-password your-password
