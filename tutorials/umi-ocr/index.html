软件简介

Umi-OCR 是一款开源免费的离线 OCR 文字识别工具，基于百度 PaddleOCR 引擎（同时也支持 RapidOCR 轻量引擎）。它完全在本地运行，不需要联网，所有图片数据不会上传到任何服务器。这意味着没有隐私泄露风险，也没有识别次数和字符数限制。

Umi-OCR 支持截图识别、批量图片识别、PDF 文字提取三大核心场景。识别引擎使用深度学习模型，对中英日韩等语言的印刷体文字有很高的准确率。软件为绿色版，下载解压即可使用，无需安装。

该软件由国内开发者 hiroi-sora 开发维护，GitHub 星标超过 27k。

核心特性

完全离线：本地识别，不上传任何数据，保护隐私安全

截图识别：快捷键截图即识别，复制文字一气呵成

批量识别：支持拖拽多张图片到窗口批量处理

PDF 识别：从 PDF 文件中提取文字，支持多页

多国语言：支持简繁体中文、英文、日文、韩文、俄文、法文、德文等

段落合并：自动合并同一段落内的文字行，保持阅读顺序

多种输出：复制到剪贴板、保存为 txt / md / json 格式

二维码识别：识别图片中的二维码和条形码内容

开源免费：MIT 协议，无任何收费限制

下载与安装

### 下载方式

从 Umi-OCR 官方 GitHub Releases 页面下载：

https://github.com/hiroi-sora/Umi-OCR/releases

Windows 用户下载 Umi-OCR_版本号_Windows.zip（约 30-60MB，含简体中文识别库的完整包约 120MB）

区别说明：

完整包（含简体中文识别库）：开箱即用，包含简体中文语言包，适合只需要识别中文的用户

解耦包：不包含识别库，需要手动下载语言包放到 `PaddleOCR-json` 目录下，适合需要自定义语言的用户

### 安装步骤

Windows 版（绿色免安装）

1 解压下载的 zip 文件到任意目录（如 `D:\Umi-OCR`）

2 进入解压目录，双击运行 `Umi-OCR.exe` 即可

3 建议右键点击 `Umi-OCR.exe`，选择"发送到 > 桌面快捷方式"，方便以后启动

> 注意：第一次启动时可能会被 Windows Defender 或杀毒软件拦截。这是由于 PaddleOCR 引擎的深度学习模型文件被部分安全软件误报。请放心添加信任/允许运行，软件是开源的无毒程序。

macOS / Linux 版

Umi-OCR 主要为 Windows 平台开发，macOS 和 Linux 用户可以使用 V2 版本，但功能和稳定性不如 Windows 版。

截图识别（最常用的功能）

### 基础操作

1 启动 Umi-OCR，主窗口会在屏幕右上角显示

2 默认快捷键 `Ctrl+Shift+Alt+S` 激活截图（也可以点击主窗口的 截图识别 按钮）

3 屏幕变暗，鼠标拖拽选择要识别的区域

4 松开鼠标后，区域内的文字会被自动识别

5 识别结果会显示在主窗口的文本框中

6 识别后的文字会自动复制到剪贴板（可在设置中关闭）

### 截图识别快捷键

默认快捷键：

截图识别：`Ctrl + Shift + Alt + S`

局部截图识别：`Ctrl + Shift + Alt + A`（从已有截图中二次框选）

### 截图模式

Umi-OCR 的截图识别有几种模式，可以在设置中切换：

默认模式：选择区域后立即识别并返回结果

合并模式（推荐）：选择区域后，先不退出截图界面，可以多次框选，识别结果会自动合并。适合需要从屏幕不同位置提取多段文字的场景

拖拽文件模式：鼠标拖拽外部图片文件到 Umi-OCR 窗口，自动识别

批量识别

### 图片批量识别

1 在 Umi-OCR 主界面切换到 批量识别 标签页

2 可以直接拖拽图片文件或文件夹到灰色拖拽区域

3 也可以点击 添加图片 或 添加文件夹 按钮选择文件

4 支持格式：png、jpg、jpeg、bmp、tiff、webp

5 点击 开始任务 按钮

6 进度条会显示识别进度，完成后所有结果会显示在列表中

7 点击每条结果可以查看识别详情，点击右下角按钮可以一键导出

### 输出格式

批量识别结果支持三种输出格式：

TXT：纯文本格式，每张图片的结果连续排列

MD：Markdown 格式，每张图片的结果用 Markdown 块引用分隔

JSON：结构化数据，包含每张图片的识别文本、坐标（文字位置）、置信度等信息，适合程序化处理

也可以在识别前勾选 复制结果到剪贴板，识别完成后直接粘贴使用。

PDF 文字提取

1 在主界面切换到 批量识别 标签页

2 拖拽或添加 PDF 文件（支持多页 PDF）

3 软件会自动将 PDF 的每一页转换为图片，然后进行 OCR 识别

4 识别完成后可以导出为 TXT / MD / JSON

> 注意：扫描版 PDF（即由扫描仪生成的图片式 PDF）是 Umi-OCR 的主要应用场景。而对于本身就已经包含文字层的 PDF（数字生成的 PDF），建议直接用 PDF 阅读器复制文字即可，不需要使用 OCR。

### PDF 识别设置

在设置中可以调整 PDF 识别相关参数：

DPI：PDF 转图片的分辨率，默认 200。值越大图片越清晰，识别率越高，但处理速度变慢

输出编码：默认 UTF-8，按需选择

语言模型切换

Umi-OCR 支持切换不同的识别语言和识别引擎。

### 切换语言

在主界面的识别语言下拉框中可以选择：

简体中文（默认，识别率最高）

繁体中文

英文

日文

韩文

俄文

多语言混合：适合中英文混排的文档

其他语言需要额外下载语言包

> 提示：如果文档是纯中文，使用"简体中文"识别率最高。如果文档中英混排，使用"简体中文+英文"混合模式效果最好。

### 引擎选择

Umi-OCR 支持两种 OCR 引擎：

PaddleOCR 引擎（默认）

识别率更高，尤其是中文识别

模型文件较大（约 30-100MB）

对硬件有一定要求，但普通 CPU 即可流畅运行

RapidOCR 引擎

更加轻量，启动速度快

模型文件小（约 10MB）

识别率略低于 PaddleOCR

适合对启动速度有要求的场景

在设置中可以选择使用的引擎。下载解耦版后需要手动下载对应的引擎包。

输出格式设置

在 设置 > 输出 中可以配置：

自动复制到剪贴板：截图识别完成后自动将文字复制到剪贴板，默认开启

输出格式：纯文本 / 文本+坐标

去除换行符：将识别结果中的换行符替换为空格，适合一段连贯的文字

段落合并：智能合并同一段落内的文字行，保持原文的段落结构

输出编码：UTF-8 / ANSI

### 段落合并设置

段落合并是 Umi-OCR 的一个实用功能：

开启后，识别结果会根据文字位置自动判断段落边界

同一段落内的多行文字会合并为一个段落

段落之间用空行分隔，保持原文的排版结构

对于多列排版的文档，建议关闭段落合并以避免文字顺序错乱

快捷键自定义

在 设置 > 快捷键 中，可以自定义所有快捷键：

| 功能 | 默认快捷键 |

|------|-----------|

| 截图并识别 | Ctrl + Shift + Alt + S |

| 局部截图（二次框选） | Ctrl + Shift + Alt + A |

| 截图模式下切换合并模式 | Tab |

| 取消截图 | Esc |

| 确认截图 | Enter / 空格 |

| 停止识别任务 | Ctrl + Shift + Alt + Q |

可以修改为任意组合键，避免与系统中的其他快捷键冲突。

使用技巧

### 局部截图识别

如果截取的图片区域中只有一部分需要识别：

1 按下截取大范围区域

2 在截图界面中，再次鼠标拖拽选取需要识别的局部区域（默认快捷键 A）

3 按 Enter 或空格键确认，只会识别二次选中的区域

### 多列文字识别设置

对于报纸、杂志等分栏排列的文档：

1 关闭 段落合并 功能

2 已关闭段落合并的情况下，识别结果会按从左到右、从上到下的顺序逐行输出

3 如果输出顺序混乱，可以尝试切换到 RapidOCR 引擎，其对多列排版的识别顺序可能更好

### 提升识别准确率的小技巧

1 保持图片清晰：截图时尽量放大内容，让小字变大字

2 避免倾斜：尽量让文字水平，倾斜的文字识别率会下降

3 调整 DPI：对 PDF 识别时，如果文字较小可以提高到 300 DPI

4 选择合适的语言模型：中英文混排选"简体中文+英文"，纯中文选"简体中文"

5 使用二值化预处理：在批量识别的图片设置中，可以开启二值化处理，提高低对比度图片的识别率

6 适当裁剪：去掉图片中无关的背景和装饰元素

常见问题

Q：识别结果出现乱码？

A：检查识别语言设置是否正确。如果识别的文档是简体中文但语言模型设置成了英文，会出现大量乱码。另外可以尝试切换到 PaddleOCR 引擎。

Q：杀毒软件报毒/拦截？

A：Umi-OCR 包含深度学习模型库（DLL 文件），部分杀毒软件会误报。请到 GitHub Releases 页面核对文件哈希值，一致则说明文件安全完整，添加到杀毒软件信任列表即可。

Q：点击截图后屏幕没有变暗，无法截图？

A：这通常是因为系统上安装的截图工具（如微信截图、Snipaste 等）拦截了快捷键。尝试：在 Umi-OCR 设置中修改快捷键；或者关闭其他占用 PrtSc / 截图快捷键的软件。

Q：PDF 识别速度很慢？

A：PDF 识别需要先将每页转换为图片再进行 OCR，页数较多时确实需要等待。可以降低 DPI 到 150 来提高速度。对于超大 PDF，建议拆分成小文件分批识别。

Q：支持识别手写文字吗？

A：Umi-OCR 默认的识别模型主要针对印刷体。手写体识别率不高。如果确实需要手写识别，可以尝试训练自定义模型或使用专门的手写体识别工具。

Q：Umi-OCR 支持 Mac/Linux 吗？

A：Umi-OCR V2 版本提供 macOS 和 Linux 支持，但功能和稳定性以 Windows 版最佳。macOS 版需要手动安装 Python 依赖环境。
