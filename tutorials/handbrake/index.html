入门软件：HandBrake

从基础转换到高级压缩，手把手教你用 HandBrake 处理各种视频格式。

下载与安装

1 打开官网 handbrake.fr，点击首页的 "Download HandBrake" 按钮。

2 选择对应的操作系统版本：

- Windows：下载 .exe 安装包，双击运行，一路 Next 完成安装。

- macOS：下载 .dmg 文件，将 HandBrake 拖入 Applications 文件夹。

- Linux：通过 Flatpak 安装 flatpak install handbrake，或使用包管理器 sudo apt install handbrake（Ubuntu/Debian）。

3 安装完成后启动 HandBrake，主界面分为三部分：左侧是源选择，中间是预览窗口，右侧是编码参数设置面板。

> 包管理器安装：Windows 用户可用 winget install HandBrake.HandBrake，macOS 用户可用 brew install --cask handbrake。

界面概览

HandBrake 的界面设计清晰，所有操作都在一个窗口中完成：

顶部工具栏：打开文件/文件夹、添加到队列、开始编码、预览按钮。

左侧面板：预设选择区，内置大量设备预设。

中间区域：视频预览窗口，显示当前设置的效果。

右侧面板：参数选项卡——摘要、视频、音频、字幕、滤镜、章节、高级。

预设系统：一键适配各种设备

HandBrake 内置 80+ 预设模板，覆盖常见使用场景：

### 常用预设分类

| 分类 | 适用场景 | 推荐预设 |

|------|---------|---------|

| Web 发布 | 上传 B站/YouTube/抖音 | Web > YouTube 1080p 或 Web > Vimeo 1080p |

| 通用兼容 | 分享给他人，兼容性优先 | Gmail > 25MB 限制（自动压缩到 25MB 以内） |

| 移动设备 | iPhone/iPad/安卓手机 | Devices > Apple 1080p 或 Android 1080p |

| 电视/游戏机 | PS4/PS5/Xbox/Smart TV | Playstation 4 > 1080p 等 |

| 无损存档 | 个人存档保留最高画质 | Production > Standard |

| Discord 分享 | 压缩到 Discord 8MB 上传限制 | Discord > 8MB 限制 |

自定义预设：调整好参数后，点击工具栏的"保存预设"按钮，输入名称即可保存你自己的预设。

视频编码器详解

HandBrake 支持多种视频编码器，选择合适的编码器是平衡画质和文件大小的关键。

### 软件编码器

| 编码器 | 压缩率 | 兼容性 | 速度 | 适用场景 |

|--------|--------|--------|------|---------|

| H.264 (x264) | 中等 | 最好 | 快 | 最通用，几乎所有设备都能播放 |

| H.265 (x265) | 高（比 H.264 省 30-50%） | 良好 | 较慢 | 4K 视频压缩，追求小体积 |

| AV1 (SVT-AV1) | 最高（比 H.265 再省 30%） | 一般 | 很慢 | 追求极致压缩率，新设备支持 |

| VP9 | 高 | 良好 | 慢 | WebM 格式，适合网页 |

选择建议：

日常分享 → H.264（兼容性最好）

4K 视频压缩 → H.265（画质/体积平衡最佳）

上传 YouTube → H.264（YouTube 转码后画质损失最小）

个人存档 → H.265 或 AV1（省空间）

### 硬件编码器（速度更快，但画质略低）

| 编码器 | 适用显卡 | 速度优势 |

|--------|---------|---------|

| NVIDIA NVENC | NVIDIA GTX 1060+ | 比 CPU 快 5-10 倍 |

| Intel QSV | Intel 核显 (6代+) | 几乎不影响 CPU 占用 |

| AMD VCN | AMD RX 5000+ 系列 | 速度快功耗低 |

| Apple VideoToolbox | Mac (Apple Silicon/Intel) | macOS 原生加速 |

> 小技巧：如果时间不急，用软件编码器画质更好。如果需要批量处理大量视频，用硬件编码器可大幅节省时间。

质量设置：CRF vs 码率

HandBrake 提供两种质量控制方式：

### CRF（恒定质量）— 推荐

CRF 是最直观的质量控制方式：

| CRF 值 | 质量 | 文件大小 |

|--------|------|---------|

| 18-20 | 视觉无损 | 较大 |

| 21-23 | 高质量（推荐） | 适中 |

| 24-26 | 中等 | 较小 |

| 27-30 | 低质量 | 很小 |

建议范围：H.264 用 CRF 20-22，H.265 用 CRF 22-24。

### 平均码率（ABR）— 精确控制大小

适用于需要精确控制输出文件大小的场景（如上传到有大小限制的平台）。

滤镜功能

HandBrake 内置多种视频滤镜，可在 滤镜 选项卡中设置：

去隔行（Deinterlace）：去除老旧隔行扫描视频的扫描线。

去噪（Denoise）：降低画面噪点，可选用 hqdn3d 或 NLMeans。

锐化（Sharpen）：提升画面清晰度，适合模糊的源视频。

裁剪（Crop）：自动或手动去除黑边。

缩放（Scale）：更改视频分辨率。

色彩空间：调整色彩参数。

字幕处理

HandBrake 支持多种字幕处理方式：

1 在 字幕 选项卡中点击"添加字幕"。

2 选择 内嵌字幕（从源文件中提取）或 外部字幕（加载 .srt/.ass 文件）。

3 选择 烧录（Burn In）：字幕永久嵌入画面，不可关闭。

4 或选择 软字幕：作为独立轨道，播放时可开关。

> 分享给他人时建议使用软字幕，保留切换能力。

批量处理与队列

HandBrake 的队列功能让批量处理变得简单：

1 打开第一个视频文件，设置好编码参数。

2 点击工具栏的 添加到队列 按钮。

3 打开第二个视频文件，设置参数，再次添加到队列。

4 重复以上步骤将所有文件添加到队列。

5 点击 开始编码，HandBrake 会依次处理所有队列任务。

批量标签（Batch Titles）：在 摘要 → 输出文件名 中输入 {source} 作为动态文件名，自动匹配源文件名。

命令行 CLI 进阶用法

HandBrake 命令行版本（HandBrakeCLI）适合自动化脚本：

HandBrakeCLI -i input.mp4 -o output.mp4 -Z "YouTube 1080p"

HandBrakeCLI -i input.mkv -o output.mp4 -e x264 -q 22

HandBrakeCLI -i input.mp4 -o output.mp4 --start-at seconds:30 --stop-at seconds:60

for f in *.mkv; do

HandBrakeCLI -i "$f" -o "${f%.mkv}.mp4" -Z "Fast 1080p30"

done

常见问题

### HandBrake 转换后的视频画质变差了？

检查 CRF 值是否设置过低（数值越高画质越差）。建议 H.264 使用 CRF 20-22，保持接近无损的画质。如果使用硬件编码器（NVENC/QSV），画质会比软件编码器略差，这是正常的。

### 转换速度太慢怎么办？

1 降低预设等级：从 `Slow` 改为 `Medium` 或 `Fast`，速度显著提升，画质差异很小。

2 启用硬件编码器（NVENC/QSV/AMD VCN），速度可提升 5-10 倍。

3 关闭不必要的滤镜（如去噪、锐化会大幅降低速度）。

4 确保 CPU 温度正常，散热不足会导致降频。

### 转换后的文件比原文件还大？

1 使用了过低的 CRF 值（如 16-18），尝试提高到 22-24。

2 原视频已经是高压缩格式（如 H.265 或 HEVC），重新压缩效果有限。

3 码率模式设置过高，切换到 CRF 模式会更合理。

### 支持哪些输入格式？

HandBrake 支持几乎所有常见视频格式和编码，包括 MP4、MKV、AVI、MOV、WMV、FLV 以及 DVD/蓝光光盘。同时支持消费级摄像机、手机拍摄、游戏录屏等来源。

### 怎么保留视频的 HDR 效果？

在 视频 选项卡中，选择 H.265 或 AV1 编码器，勾选 "HDR 编码" 相关选项。HandBrake 支持 HDR10 和 HDR10+ 的元数据传递，确保转换后 HDR 效果完整保留。
