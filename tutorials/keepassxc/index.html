安装方式

### Windows

安装包下载：访问 [KeePassXC 官网下载页](https://keepassxc.org/download/#windows) 下载安装包。

winget：

winget install KeePassXC.KeePassXC

Chocolatey：

choco install keepassxc

Portable 版：下载 .zip 压缩包，解压后直接运行 KeePassXC.exe，无需安装。

### macOS

Homebrew：

brew install --cask keepassxc

手动安装：从官网下载 .dmg 文件，拖入 Applications 文件夹。

### Linux

Ubuntu/Debian：

sudo apt install keepassxc

Fedora：

sudo dnf install keepassxc

Arch Linux：

sudo pacman -S keepassxc

Flatpak：

flatpak install flathub org.keepassxc.KeePassXC

创建第一个密码数据库

### 第一步：新建数据库

1 启动 KeePassXC，点击 "创建新数据库" 或菜单栏 > 数据库 > 新建。

2 选择保存路径（建议放在云盘同步目录中，如 OneDrive/Dropbox/Nextcloud），文件名如 `MyPasswords.kdbx`。

3 点击"继续"进入加密设置。

### 第二步：设置主密码

1 在"加密设置"页面：

- 加密算法：选择 AES-256（兼容性最好）或 ChaCha20（速度快，适合老设备）。

- 解密轮数：默认 1 秒延迟（越久越安全，推荐保留默认）。

2 在"主密码"页面输入密码：

- 推荐长度：至少 12 位，包含大小写字母、数字和符号。

- 密码强度指示条变成绿色即表示强度足够。

- 一定要记住这个密码！ 丢失主密码 = 永久失去所有密码，没有人能帮你找回。

> 可选：绑定硬件密钥（YubiKey / OnlyKey）作为第二解锁因子，需插入硬件密钥才能打开数据库（详情见"进阶设置"部分）。

### 第三步：数据库设置

点击"完成"后，建议立即配置：

数据库名称：如"我的密码库"

默认用户名：填写你最常用的用户名/邮箱，新增条目时自动填充。

回收站：勾选"启用回收站"（删除的条目先进入回收站而非永久删除）。

自动保存：推荐取消"在保存时询问"的勾选，让 KeePassXC 自动保存。

添加和管理密码条目

### 添加新条目

1 在主界面左侧选择一个分组（默认有"根"组）。

2 点击工具栏的 "添加条目" 按钮（+ 图标）或按 `Ctrl+N`。

3 填写条目信息（必填仅标题和密码，其他可选）：

- 标题：如"Google 账户"

- 用户名：你的登录邮箱或用户名

- 密码：点击右侧骰子图标可打开密码生成器

- URL：网站地址（浏览器集成自动填充时使用）

- 备注：安全提示问题、PIN 码等额外信息

4 点击 "保存条目"。

### 分组管理

右键左侧的分组可进行管理：

新建分组：按类别创建（如"银行"、"社交媒体"、"工作"、"邮箱"）。

拖拽移动：将条目拖拽到不同分组即可归类。

颜色标记：右键分组 > 设置分组颜色，方便视觉区分。

子分组：支持无限层级嵌套，如"工作 > 公司邮箱 > 项目账号"。

### 搜索条目

在主界面搜索框输入关键词：

搜索范围为所有条目（不限于当前分组）。

支持搜索标题、用户名、URL、备注所有字段。

点击搜索框左侧的漏斗图标可设置高级搜索条件。

密码生成器

KeePassXC 内置强大的密码生成器，可生成高强度且易读的密码。

### 打开密码生成器

方法一：添加/编辑条目时，点击密码输入框右侧的 骰子图标。

方法二：工具菜单 > 密码生成器（`Ctrl+G`）。

### 生成模式

1. 标准密码（适合大多数网站）

长度：推荐 16~20 位

勾选：大写字母、小写字母、数字

建议勾选：特殊字符（`!@#$%^&*`）

可选：排除相似字符（如 `Il1O0`）

2. 可读密码（Diceware 模式）

使用常用英文单词组合，如 `Correct-Horse-Battery-Staple`

推荐 4~6 个单词，用分隔符连接

优势：容易手动输入，依然高强度

适合：在手机/游戏主机等不方便粘贴密码的设备上使用

3. PIN 码

纯数字，长度 4~8 位的 PIN 码

适合：手机锁屏、银行卡 PIN

点击密码右侧的复制按钮即可将生成的密码复制到剪贴板（15 秒后自动清除）。

截图描述：密码生成器对话框，左侧选择生成模式（标准/可读密码/PIN），右侧显示生成的密码预览，下方有长度滑块、各字符类型勾选框、排除相似字符等选项，底部有"复制"和"关闭"按钮。

浏览器集成（自动填充密码）

KeePassXC 通过浏览器扩展实现自动填充，比浏览器自带密码管理器更安全和便利。

### 安装扩展

| 浏览器 | 扩展名称 | 安装地址 |

|--------|---------|---------|

| Chrome/Chromium | KeePassXC-Browser | Chrome Web Store 搜索 |

| Firefox | KeePassXC-Browser | Firefox Add-ons 搜索 |

| Edge | KeePassXC-Browser | Edge 加载项中搜索 |

| Brave/Vivaldi | KeePassXC-Browser | 通过 Chrome Web Store 安装 |

### 连接配置

1 在 KeePassXC 中：进入 设置 > 浏览器集成，勾选要连接的浏览器，点击"确定"。

2 在浏览器扩展中：点击扩展图标，会显示"KeePassXC 未连接"。

3 配对：

- 在 KeePassXC 中点击 工具 > 设置 > 浏览器集成 > "高级"选项卡。

- 点击"重新连接 KeePassXC 数据库"。

- 浏览器扩展图标显示为绿色对勾，表示连接成功。

4 首次使用：浏览器扩展会请求访问 KeePassXC 数据库，点击"允许"。

### 自动填充

1 打开任意登录页面（如 `google.com` 登录页）。

2 确保 KeePassXC 中已有该网站的条目（URL 字段需匹配当前网站域名）。

3 点击登录框，浏览器扩展图标会显示数字角标（表示当前页面有可用的凭证数量）。

4 点击浏览器扩展图标，选择对应的账户条目。

5 用户名和密码自动填入（也可以使用快捷键 `Ctrl+Shift+U` 自动填充）。

匹配规则说明：

URL 填写 `https://example.com` 可匹配 `example.com` 下所有页面。

URL 填写 `https://example.com/login` 仅匹配该具体页面。

建议使用通配符 `http*://*.example.com/*` 配整个站点的所有子域名和页面。

### 手动触发填充

如果自动填充未触发，可：

右键输入框 > KeePassXC > 用户名和密码。

或点击 KeePassXC-Browser 扩展图标手动选择条目。

或按 `Ctrl+Shift+U` 快捷键自动填充匹配的条目。

TOTP 两步验证

KeePassXC 支持内置的 TOTP（Time-based One-Time Password）生成器，无需额外安装 Google Authenticator 等应用。

### 添加 TOTP

方法一（手动输入密钥）：

1 编辑目标条目。

2 点击"高级"选项卡 > "添加 TOTP"。

3 输入服务商提供的 TOTP 密钥（通常是 Base32 编码的字符串，如 `JBSWY3DPEHPK3PXP`）。

4 点击"确定"完成。

方法二（扫描二维码）：

1 在网站的 TOTP 设置页面显示二维码时，截图保存。

2 在 KeePassXC 中点击"添加 TOTP" > "从二维码扫描"。

3 选择截图文件，自动解析二维码中的密钥。

### 使用 TOTP

查看条目详情时，TOTP 代码会自动显示并每 30 秒刷新。

双击 TOTP 码可复制到剪贴板（15 秒后自动清除）。

浏览器集成填充时，如果条目含有 TOTP，扩展会自动提示复制验证码。

### 将现有 Authenticator 迁移到 KeePassXC

1 在 Google Authenticator / Authy 中找到导出或获取密钥的功能。

2 有些服务商在两步验证设置页面会显示"无法扫描？"链接，点击可查看密钥文本。

3 将密钥手动输入或通过二维码导入 KeePassXC。

SSH Agent

KeePassXC 可以作为 SSH Agent 运行，管理你的 SSH 私钥，免去每次输入密码短语的麻烦。

### 设置步骤

1 打开 工具 > 设置 > SSH Agent。

2 勾选"启用 SSH Agent 集成"。

3 点击"添加密钥"，选择你的 SSH 私钥文件（如 `~/.ssh/id_rsa`）。

4 如果私钥有密码短语，KeePassXC 会存储在数据库中，解锁数据库时自动加载。

5 勾选"当数据库解锁时将此密钥添加到代理"。

配置完成后，打开终端执行 ssh-add -l 可看到 KeePassXC 提供的密钥列表。

硬件密钥（YubiKey 支持）

### 配置 YubiKey 作为第二因子

1 在创建新数据库或编辑现有数据库时，进入"加密设置"。

2 勾选"使用硬件密钥（YubiKey / OnlyKey）进行加密"。

3 插入 YubiKey，点击"检测 YubiKey"。

4 选择密钥槽（Slot 1 或 Slot 2），点击"配置"。

5 后续打开数据库时需同时输入主密码和插入 YubiKey 并触摸其按钮。

> YubiKey 是第二因子（2FA），不是主密码的替代品。丢失 YubiKey 仍可通过主密码打开数据库。

数据库同步（配合云盘使用）

KeePassXC 的数据库是一个本地文件，你可以通过云盘在多台设备间同步：

1 将 `.kdbx` 文件保存在云盘同步文件夹（OneDrive / Google Drive / Dropbox / Nextcloud / Synology Drive）中。

2 在每台设备上安装 KeePassXC，打开同一个 `.kdbx` 文件。

3 在一台设备上修改密码后，KeePassXC 会自动保存，云盘同步后另一台设备可打开最新版本。

同步注意事项：

不要在同一时间在多台设备上修改数据库——这会导致冲突。云盘也可能产生冲突文件（如 `MyPasswords_conflict_2025-01-01.kdbx`）。

如果发生冲突，KeePassXC 提供了 数据库合并 功能：文件 > 合并数据库，选择冲突文件，KeePassXC 会自动合并两个版本中的差异。

移动端可使用 KeePassDX（Android）或 KeePassium（iOS，付费功能支持完整编辑）搭配云盘访问同一个 `.kdbx` 文件。

导出与备份

### 备份数据库

直接复制 `.kdbx` 文件即可完成备份。

建议定期将数据库文件备份到外部存储或第二云盘。

在 KeePassXC 中点击 文件 > 保存备份为，可创建当前数据库的备份副本。

### 导出为其他格式

文件 > 导出 > 选择格式：

CSV：可用于导入其他密码管理器（但 CSV 无加密，导出后需妥善处理）。

HTML：生成一个可打印的密码列表（适合纸质备份）。

XML：完整的数据结构导出（包括自定义字段等）。

进阶设置

### 数据库安全设置

在 工具 > 设置 > 安全：

自动锁定：推荐设置为"最小化时锁定"和"暂停时锁定"。

自动锁定时间：推荐 5 分钟无操作后自动锁定（防止他人趁你离开时查看密码）。

剪贴板清除：推荐设置为 15 秒，复制密码后自动清除剪贴板内容，防止密码残留在内存中。

浏览器集成加密：勾选"使用数据库加密的浏览器集成"（增强浏览器扩展通信的安全性）。

### 数据库文件的安全建议

1 主密码使用 Diceware 模式生成的短语，长度至少 6 个单词。

2 配合 YubiKey 作为第二因子。

3 在加密设置中将解密轮数增加到 2~3 秒（攻击者破解难度翻倍）。

4 千万不要将主密码保存在任何数字设备上——手写抄下来放在安全的地方（如保险柜）。

常见问题

Q：忘记主密码怎么办？

A：KeePassXC 没有"找回密码"功能，这是设计使然（你的密码只有你知道）。如果忘记主密码，没有任何方法可以恢复数据库。请务必牢记主密码，或考虑将主密码拆分成两部分用不同方式记忆。

Q：能不能在手机上用？

A：可以，但 KeePassXC 本身只有桌面版。移动端推荐使用：

Android：[KeePassDX](https://www.keepassdx.com/)（免费开源，支持生物识别解锁）

iOS：[KeePassium](https://keepassium.com/)（免费版可查看，付费版支持编辑和创建）

只需将 .kdbx 文件通过云盘同步到手机即可。

Q：KeePassXC 和 1Password / Bitwarden 比怎么样？

A：KeePassXC 的最大优势是完全本地——不上传任何数据到云端，隐私最好。Bitwarden 优点是全平台无缝同步（包括移动端），支持云端自动备份。1Password 界面最精致但收费且使用私有格式。如果你注重隐私并愿意自行管理同步，KeePassXC 是最佳选择。

Q：浏览器集成安全吗？

A：KeePassXC 与浏览器扩展的通信使用本地 Socket 加密通信（通过配对的密钥文件），数据不会经过任何第三方服务器。每次解锁数据库时浏览器扩展自动获取访问权限，锁定后自动断开。

Q：如何修改主密码？

A：文件 > 修改数据库凭据... > 输入当前密码 > 输入新密码 > 确认。

Q：为什么我打开某些 .kdbx 文件时提示格式错误？

A：KeePassXC 使用 .kdbx 格式（KeePass 2.x），无法打开旧的 .kdb 格式（KeePass 1.x）。如果是从旧版切换，需先在 KeePass 1.x 中导出为 .kdbx。

Q：数据库文件有多大？

A：100 个条目的数据库约 10~20KB，1000 个条目约 100~200KB。非常小巧，同步和备份几乎没有负担。
